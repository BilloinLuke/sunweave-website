### 三次修复 Gitee 日志 400 "文件名已存在"（真正根因）
- 现象：用户重启后聊天可用，但 `server.log` 仍报 `[chatlog] retry failed: PUT 400 {"message":"文件名已存在"}`。
- 调试日志显示：直接读取 `chat-logs/<ip>.csv` 返回 200，已正确拿到 `content` 和 `sha`，所以不是读取失败。
- 真正根因：`giteePut()` 对更新操作使用了 `POST` 方法。Gitee Contents API 中，**POST 用于新建文件**，**PUT 用于更新文件**。带 SHA 的更新请求用 POST 发出时，Gitee 仍按"新建"处理，发现文件名已存在就返回 `400 文件名已存在`。
- 修复 `api/chatlog.js`：
  - `giteePut()` 根据是否有 `sha` 自动选择方法：`sha ? 'PUT' : 'POST'`
  - 错误信息同步改为真实方法名，方便调试
- 保留前一轮改动（`giteeGetFile` / `giteeGetDirFile` / `giteeGetRaw` / `loadEntry` / 调试日志）作为兜底和诊断能力。
- 校验：沙箱内 `node -e` 加载 `api/chatlog.js` 通过。
- 待验证：用户 `./restart.sh` 重启后，发消息看 `server.log` 是否出现 `chat log <ip> ok`，Gitee CSV 是否追加新行。
